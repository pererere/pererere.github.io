{\footnotesize\ttfamily \label{classgyp_1_1xcodeproj__file_1_1_x_c_project_file_a81a6f17d495377e9ce1fdbfd5b8e7a2a} 
gyp.\+xcodeproj\+\_\+file.\+XCProject\+File.\+Compute\+IDs (\begin{DoxyParamCaption}\item[{}]{self}{, }\item[{}]{recursive}{ = {\ttfamily True}, }\item[{}]{overwrite}{ = {\ttfamily True}, }\item[{}]{seed\+\_\+hash}{ = {\ttfamily None}}\end{DoxyParamCaption})}

\begin{DoxyVerb}Set "id" properties deterministically.

An object's "id" property is set based on a hash of its class type and
name, as well as the class type and name of all ancestor objects.  As
such, it is only advisable to call ComputeIDs once an entire project file
tree is built.

If recursive is True, recurse into all descendant objects and update their
hashes.

If overwrite is True, any existing value set in the "id" property will be
replaced.
\end{DoxyVerb}
