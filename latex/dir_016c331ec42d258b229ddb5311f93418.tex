\doxysubsection{Detailed Description}
JSON canonicalization compliant with the \href{https://wiki.laptop.org/go/Canonical_JSON}{\texttt{ OLPC Canonical JSON specification}}.\hypertarget{README.md_autotoc_md1755}{}\doxysubsection{\texorpdfstring{Why}{Why}}\label{README.md_autotoc_md1755}
If you\textquotesingle{}re looking for \href{https://www.rfc-editor.org/rfc/rfc8785}{\texttt{ RFC 8785}} compliant JSON canonicalization there are \href{https://www.npmjs.com/package/@stratumn/canonicaljson}{\texttt{ any}} \href{https://www.npmjs.com/package/@truestamp/canonify}{\texttt{ number}} \href{https://www.npmjs.com/package/canonical-json}{\texttt{ of}} \href{https://www.npmjs.com/package/canonicalize}{\texttt{ libraries}} \href{https://www.npmjs.com/package/canonicalize-json}{\texttt{ to}} \href{https://www.npmjs.com/package/json-canonicalize}{\texttt{ choose}} \href{https://www.npmjs.com/package/another-json}{\texttt{ from}}. You should only select this library if you know that you specifically need support for the \href{https://wiki.laptop.org/go/Canonical_JSON}{\texttt{ OLPC}}-\/style of canonicalization.

One reason you might chose OLPC compliance is for interoperability with \href{https://theupdateframework.github.io/specification/latest/\#metaformat}{\texttt{ The Update Framework}} which specifically calls out OLPC as the canonicalization standard for computing signatures over TUF metadata.

The canonicalized strings generated by this library are compatible with those generated by the Python-\/based \href{https://github.com/secure-systems-lab/securesystemslib}{\texttt{ securesystemslib}} library and the Go-\/based \href{https://github.com/secure-systems-lab/go-securesystemslib}{\texttt{ go-\/securesystemslib}} library.\hypertarget{README.md_autotoc_md1756}{}\doxysubsection{\texorpdfstring{Installation}{Installation}}\label{README.md_autotoc_md1756}
