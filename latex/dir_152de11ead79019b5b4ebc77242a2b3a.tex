Prior to {\ttfamily v0.\+11.\+1}, Zone.\+js provided two distribution bundle formats in the {\ttfamily dist} folder. They were (1) {\ttfamily ES5} bundle distributed as {\ttfamily zone.\+js} and (2) {\ttfamily ES2015} bundle distributed as {\ttfamily zone-\/evergreen.\+js}. Both of these bundles were in {\ttfamily UMD} format, and are used for Angular\textquotesingle{}s differential-\/loading mechanism.

Starting with {\ttfamily v0.\+11.\+1}, Zone.\+js follows the \href{https://docs.google.com/document/d/1CZC2rcpxffTDfRDs6p1cfbmKNLA6x5O-NtkJglDaBVs}{\texttt{ Angular Package Format}}. Therefor the new Zone.\+js file layout is\+:


\begin{DoxyItemize}
\item {\ttfamily bundles}\+: {\ttfamily ES5} bundle in {\ttfamily UMD} format.
\item {\ttfamily fesm2015}\+: {\ttfamily ES5} bundle in {\ttfamily ESM} format.
\item {\ttfamily dist}\+: {\ttfamily ES5} bundle in {\ttfamily UMD} format. This directory is present to keep backward compatibility.
\end{DoxyItemize}

If you are using {\ttfamily Angular CLI}, the {\ttfamily polyfills.\+ts} file will contain\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ 'zone.js/dist/zone';}

\end{DoxyCode}


Starting with Zone.\+js {\ttfamily v0.\+11.\+1+} the import changes to\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ 'zone.js';}

\end{DoxyCode}


Prior to {\ttfamily v0.\+11.\+1} the import would load the {\ttfamily ES5} bundle in {\ttfamily UMD} format from {\ttfamily dist/zone.\+js}. Starting with {\ttfamily v0.\+11.\+1} the import loads the {\ttfamily ES2015} bundle in {\ttfamily ESM} format instead.

This is a breaking change for legacy browsers such as {\ttfamily IE11}.

For backwards compatibility {\ttfamily zone.\+js} continues to distribute the same bundles under {\ttfamily dist}. To restore the old behavior import from the {\ttfamily dist} directory instead like so\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{import\ 'zone.js/dist/zone';}

\end{DoxyCode}
