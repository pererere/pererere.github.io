\doxysubsection{Detailed Description}
localtunnel exposes your localhost to the world for easy testing and sharing! No need to mess with DNS or deploy just to have others test out your changes.

Great for working with browser testing tools like browserling or external api callback services like twilio which require a public url for callbacks.\hypertarget{README.md_autotoc_md13351}{}\doxysubsection{\texorpdfstring{Quickstart}{Quickstart}}\label{README.md_autotoc_md13351}

\hypertarget{README.md_autotoc_md13355}{}\doxysubsection{\texorpdfstring{CLI usage}{CLI usage}}\label{README.md_autotoc_md13355}
When localtunnel is installed globally, just use the {\ttfamily lt} command to start the tunnel.


\begin{DoxyCode}{0}
\DoxyCodeLine{lt\ -\/-\/port\ 8000}

\end{DoxyCode}


Thats it! It will connect to the tunnel server, setup the tunnel, and tell you what url to use for your testing. This url will remain active for the duration of your session; so feel free to share it with others for happy fun time!

You can restart your local server all you want, {\ttfamily lt} is smart enough to detect this and reconnect once it is back.\hypertarget{README.md_autotoc_md13356}{}\doxysubsubsection{\texorpdfstring{Arguments}{Arguments}}\label{README.md_autotoc_md13356}

The localtunnel client is also usable through an API (for test integration, automation, etc)\hypertarget{README.md_autotoc_md13358}{}\doxysubsubsection{\texorpdfstring{localtunnel(port \mbox{[},options\mbox{]}\mbox{[},callback\mbox{]})}{localtunnel(port \mbox{[},options\mbox{]}\mbox{[},callback\mbox{]})}}\label{README.md_autotoc_md13358}
Creates a new localtunnel to the specified local {\ttfamily port}. Will return a Promise that resolves once you have been assigned a public localtunnel url. {\ttfamily options} can be used to request a specific {\ttfamily subdomain}. A {\ttfamily callback} function can be passed, in which case it won\textquotesingle{}t return a Promise. This exists for backwards compatibility with the old Node-\/style callback API. You may also pass a single options object with {\ttfamily port} as a property.

\hypertarget{README.md_autotoc_md13359}{}\doxysubsubsubsection{\texorpdfstring{options}{options}}\label{README.md_autotoc_md13359}

\begin{DoxyItemize}
\item {\ttfamily port} (number) \mbox{[}required\mbox{]} The local port number to expose through localtunnel.
\item {\ttfamily subdomain} (string) Request a specific subdomain on the proxy server. {\bfseries{Note}} You may not actually receive this name depending on availability.
\item {\ttfamily host} (string) URL for the upstream proxy server. Defaults to {\ttfamily \href{https://localtunnel.me}{\texttt{ https\+://localtunnel.\+me}}}.
\item {\ttfamily local\+\_\+host} (string) Proxy to this hostname instead of {\ttfamily localhost}. This will also cause the {\ttfamily Host} header to be re-\/written to this value in proxied requests.
\item {\ttfamily local\+\_\+https} (boolean) Enable tunneling to local HTTPS server.
\item {\ttfamily local\+\_\+cert} (string) Path to certificate PEM file for local HTTPS server.
\item {\ttfamily local\+\_\+key} (string) Path to certificate key file for local HTTPS server.
\item {\ttfamily local\+\_\+ca} (string) Path to certificate authority file for self-\/signed certificates.
\item {\ttfamily allow\+\_\+invalid\+\_\+cert} (boolean) Disable certificate checks for your local HTTPS server (ignore cert/key/ca options).
\end{DoxyItemize}
