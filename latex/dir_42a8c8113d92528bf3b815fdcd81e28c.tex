This plugin extracts CSS into separate files. It creates a CSS file per JS file which contains CSS. It supports On-\/\+Demand-\/\+Loading of CSS and Source\+Maps.

It builds on top of a new webpack v5 feature and requires webpack 5 to work.

Compared to the extract-\/text-\/webpack-\/plugin\+:


\begin{DoxyItemize}
\item Async loading
\item No duplicate compilation (performance)
\item Easier to use
\item Specific to CSS
\end{DoxyItemize}\hypertarget{README.md_autotoc_md14030}{}\doxysubsubsection{\texorpdfstring{Getting Started}{Getting Started}}\label{README.md_autotoc_md14030}

By default, the {\ttfamily mini-\/css-\/extract-\/plugin} appends styles ({\ttfamily \texorpdfstring{$<$}{<}link\texorpdfstring{$>$}{>}} elements) to {\ttfamily document.\+head} of the current {\ttfamily window}.

However in some circumstances it might be necessary to have finer control over the append target or even delay {\ttfamily link} elements insertion. For example this is the case when you asynchronously load styles for an application that runs inside of an iframe. In such cases {\ttfamily insert} can be configured to be a function or a custom selector.

If you target an \href{https://developer.mozilla.org/en-US/docs/Web/API/HTMLIFrameElement}{\texttt{ iframe}} make sure that the parent document has sufficient access rights to reach into the frame document and append elements to it.\hypertarget{README.md_autotoc_md14037}{}\doxysubsubsubsubsubsection{\texorpdfstring{{\ttfamily string}}{{\ttfamily string}}}\label{README.md_autotoc_md14037}
