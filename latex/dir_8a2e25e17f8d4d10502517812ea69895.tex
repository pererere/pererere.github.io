\doxysubsection{Detailed Description}
\href{https://nodei.co/npm/log4js/}{\texttt{ }}

This is a conversion of the \href{https://github.com/stritti/log4js}{\texttt{ log4js}} framework to work with \href{http://nodejs.org}{\texttt{ node}}. I started out just stripping out the browser-\/specific code and tidying up some of the javascript to work better in node. It grew from there. Although it\textquotesingle{}s got a similar name to the Java library \href{https://logging.apache.org/log4j/2.x/}{\texttt{ log4j}}, thinking that it will behave the same way will only bring you sorrow and confusion.

The full documentation is available \href{https://log4js-node.github.io/log4js-node/}{\texttt{ here}}.

\href{https://log4js-node.github.io/log4js-node/v3-changes.md}{\texttt{ Changes in version 3.\+x}}

There have been a few changes between log4js 1.\+x and 2.\+x (and 0.\+x too). You should probably read this \href{https://log4js-node.github.io/log4js-node/migration-guide.html}{\texttt{ migration guide}} if things aren\textquotesingle{}t working.

Out of the box it supports the following features\+:


\begin{DoxyItemize}
\item coloured console logging to stdout or stderr
\item file appender, with configurable log rolling based on file size or date
\item a logger for connect/express servers
\item configurable log message layout/patterns
\item different log levels for different log categories (make some parts of your app log as DEBUG, others only ERRORS, etc.)
\end{DoxyItemize}

By default, log4js will not output any logs (so that it can safely be used in libraries). The {\ttfamily level} for the {\ttfamily default} category is set to {\ttfamily OFF}. To enable logs, set the level (as in the example). This will then output to stdout with the coloured layout (thanks to \href{http://github.com/masylum}{\texttt{ masylum}}), so for the above you would see\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{[2010-\/01-\/17\ 11:43:37.987]\ [DEBUG]\ [default]\ -\/\ Some\ debug\ messages}

\end{DoxyCode}
