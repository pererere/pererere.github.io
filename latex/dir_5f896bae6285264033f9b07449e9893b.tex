\hypertarget{README.md_autotoc_md2358}{}\doxysubsubsubsection{\texorpdfstring{{\ttfamily range} and {\ttfamily exclusive\+Range}}{{\ttfamily range} and {\ttfamily exclusive\+Range}}}\label{README.md_autotoc_md2358}
Syntax sugar for the combination of minimum and maximum keywords (or exclusive\+Minimum and exclusive\+Maximum), also fails schema compilation if there are no numbers in the range.

The value of these keywords must be an array consisting of two numbers, the second must be greater or equal than the first one.

If the validated value is not a number the validation passes, otherwise to pass validation the value should be greater (or equal) than the first number and smaller (or equal) than the second number in the array.

\hypertarget{README.md_autotoc_md2361}{}\doxysubsubsubsection{\texorpdfstring{{\ttfamily transform}}{{\ttfamily transform}}}\label{README.md_autotoc_md2361}
This keyword allows a string to be modified during validation.

This keyword applies only to strings. If the data is not a string, the {\ttfamily transform} keyword is ignored.

A standalone string cannot be modified, i.\+e. `data = \textquotesingle{}a'; ajv.\+validate(schema, data);\`{}, because strings are passed by value

{\bfseries{Supported transformations\+:}}


\begin{DoxyItemize}
\item {\ttfamily trim}\+: remove whitespace from start and end
\item {\ttfamily trim\+Start}/{\ttfamily trim\+Left}\+: remove whitespace from start
\item {\ttfamily trim\+End}/{\ttfamily trim\+Right}\+: remove whitespace from end
\item {\ttfamily to\+Lower\+Case}\+: convert to lower case
\item {\ttfamily to\+Upper\+Case}\+: convert to upper case
\item {\ttfamily to\+Enum\+Case}\+: change string case to be equal to one of {\ttfamily enum} values in the schema
\end{DoxyItemize}

Transformations are applied in the order they are listed.

Note\+: {\ttfamily to\+Enum\+Case} requires that all allowed values are unique when case insensitive.

\hypertarget{README.md_autotoc_md2365}{}\doxysubsubsubsection{\texorpdfstring{{\ttfamily all\+Required}}{{\ttfamily all\+Required}}}\label{README.md_autotoc_md2365}
This keyword allows to require the presence of all properties used in {\ttfamily properties} keyword in the same schema object.

This keyword applies only to objects. If the data is not an object, the validation succeeds.

The value of this keyword must be boolean.

If the value of the keyword is {\ttfamily false}, the validation succeeds.

If the value of the keyword is {\ttfamily true}, the validation succeeds if the data contains all properties defined in {\ttfamily properties} keyword (in the same schema object).

If the {\ttfamily properties} keyword is not present in the same schema object, schema compilation will throw exception.

\hypertarget{README.md_autotoc_md2373}{}\doxysubsubsubsection{\texorpdfstring{{\ttfamily select}/{\ttfamily select\+Cases}/{\ttfamily select\+Default}}{{\ttfamily select}/{\ttfamily select\+Cases}/{\ttfamily select\+Default}}}\label{README.md_autotoc_md2373}
{\bfseries{Please note}}\+: these keywords are deprecated. It is recommended to use Open\+API \href{https://ajv.js.org/json-schema.html\#discriminator}{\texttt{ discriminator}} keyword supported by Ajv v8 instead of {\ttfamily select}.

These keywords allow to choose the schema to validate the data based on the value of some property in the validated data.

These keywords must be present in the same schema object ({\ttfamily select\+Default} is optional).

The value of {\ttfamily select} keyword should be a \href{https://github.com/ajv-validator/ajv/blob/master/docs/validation.md\#data-reference}{\texttt{ \$data reference}} that points to any primitive JSON type (string, number, boolean or null) in the data that is validated. You can also use a constant of primitive type as the value of this keyword (e.\+g., for debugging purposes).

The value of {\ttfamily select\+Cases} keyword must be an object where each property name is a possible string representation of the value of {\ttfamily select} keyword and each property value is a corresponding schema (from draft-\/06 it can be boolean) that must be used to validate the data.

The value of {\ttfamily select\+Default} keyword is a schema (also can be boolean) that must be used to validate the data in case {\ttfamily select\+Cases} has no key equal to the stringified value of {\ttfamily select} keyword.

The validation succeeds in one of the following cases\+:


\begin{DoxyItemize}
\item the validation of data using selected schema succeeds,
\item none of the schemas is selected for validation,
\item the value of select is undefined (no property in the data that the data reference points to).
\end{DoxyItemize}

If {\ttfamily select} value (in data) is not a primitive type the validation fails.

This keyword correctly tracks evaluated properties and items to work with {\ttfamily unevaluated\+Properties} and {\ttfamily unevaluated\+Items} keywords -\/ only properties and items from the subschema that was used (one of {\ttfamily select\+Cases} subschemas or {\ttfamily select\+Default} subschema) are marked as evaluated.

{\bfseries{Please note}}\+: these keywords require Ajv {\ttfamily \$data} option to support \href{https://github.com/ajv-validator/ajv/blob/master/docs/validation.md\#data-reference}{\texttt{ \$data reference}}.

\hypertarget{README.md_autotoc_md2374}{}\doxysubsubsubsection{\texorpdfstring{{\ttfamily dynamic\+Defaults}}{{\ttfamily dynamic\+Defaults}}}\label{README.md_autotoc_md2374}
This keyword allows to assign dynamic defaults to properties, such as timestamps, unique IDs etc.

This keyword only works if {\ttfamily use\+Defaults} options is used and not inside {\ttfamily any\+Of} keywords etc., in the same way as \href{https://github.com/epoberezkin/ajv\#assigning-defaults}{\texttt{ default keyword treated by Ajv}}.

The keyword should be added on the object level. Its value should be an object with each property corresponding to a property name, in the same way as in standard {\ttfamily properties} keyword. The value of each property can be\+:


\begin{DoxyItemize}
\item an identifier of dynamic default function (a string)
\item an object with properties {\ttfamily func} (an identifier) and {\ttfamily args} (an object with parameters that will be passed to this function during schema compilation -\/ see examples).
\end{DoxyItemize}

The properties used in {\ttfamily dynamic\+Defaults} should not be added to {\ttfamily required} keyword in the same schema (or validation will fail), because unlike {\ttfamily default} this keyword is processed after validation.

There are several predefined dynamic default functions\+:


\begin{DoxyItemize}
\item {\ttfamily "{}timestamp"{}} -\/ current timestamp in milliseconds
\item {\ttfamily "{}datetime"{}} -\/ current date and time as string (ISO, valid according to {\ttfamily date-\/time} format)
\item {\ttfamily "{}date"{}} -\/ current date as string (ISO, valid according to {\ttfamily date} format)
\item {\ttfamily "{}time"{}} -\/ current time as string (ISO, valid according to {\ttfamily time} format)
\item {\ttfamily "{}random"{}} -\/ pseudo-\/random number in \mbox{[}0, 1) interval
\item {\ttfamily "{}randomint"{}} -\/ pseudo-\/random integer number. If string is used as a property value, the function will randomly return 0 or 1. If object `\{ func\+: \textquotesingle{}randomint', args\+: \{ max\+: N \} \}{\ttfamily is used then the default will be an integer number in \mbox{[}0, N) interval. -\/}"{}seq"{}{\ttfamily -\/ sequential integer number starting from 0. If string is used as a property value, the default sequence will be used. If object}\{ func\+: \textquotesingle{}seq\textquotesingle{}, args\+: \{ name\+: \textquotesingle{}foo\textquotesingle{}\} \}{\ttfamily is used then the sequence with name}"{}foo"{}\`{} will be used. Sequences are global, even if different ajv instances are used.
\end{DoxyItemize}
