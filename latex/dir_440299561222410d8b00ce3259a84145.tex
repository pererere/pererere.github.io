\hypertarget{README.md_autotoc_md1999}{}\doxysubsection{\texorpdfstring{Tests}{Tests}}\label{README.md_autotoc_md1999}
This module is tested in two ways. The first one is through a small set of test cases that can be found in \href{https://github.com/maurobringolf/@webassemblyjs/floating-point-hex-parser/blob/master/test/regular.test.js}{\texttt{ test/regular.\+test.\+js}}. The second one is non-\/deterministic (sometimes called {\itshape fuzzing})\+:


\begin{DoxyEnumerate}
\item Generate a random IEEE754 double precision value {\ttfamily x}.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Compute its representation {\ttfamily y} in floating point hexadecimal format using the C standard library function {\ttfamily printf} since C supports this format.
\end{DoxyEnumerate}
\begin{DoxyEnumerate}
\item Give both values to JS testcase and see if {\ttfamily parse\+Hex\+Float(y) === x}.
\end{DoxyEnumerate}

By default one {\ttfamily npm test} run tests 100 random samples. If you want to do more, you can set the environment variable {\ttfamily FUZZ\+\_\+\+AMOUNT} to whatever number of runs you\textquotesingle{}d like. Because it uses one child process for each sample, it is really slow though.
