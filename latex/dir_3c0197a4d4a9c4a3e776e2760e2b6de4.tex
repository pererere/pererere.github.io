{\bfseries{Note\+:}} A falsy or malformed token will throw an {\ttfamily Invalid\+Token\+Error} error; see below for more information on specific errors.\hypertarget{README.md_autotoc_md11161}{}\doxysubsubsection{\texorpdfstring{Errors}{Errors}}\label{README.md_autotoc_md11161}
This library works with valid JSON web tokens. The basic format of these token is 
\begin{DoxyCode}{0}
\DoxyCodeLine{[part1].[part2].[part3]}

\end{DoxyCode}
 All parts are supposed to be valid base64 (url) encoded json. Depending on the {\ttfamily \{ header\+: \texorpdfstring{$<$}{<}option\texorpdfstring{$>$}{>} \}} option it will decode part 1 (only if header\+: true is specified) or part 2 (default)

Not adhering to the format will result in a {\ttfamily Invalid\+Token\+Error} with one of the following messages\+:


\begin{DoxyItemize}
\item {\ttfamily Invalid token specified\+: must be a string} =\texorpdfstring{$>$}{>} the token passed was not a string, this library only works on strings.
\item {\ttfamily Invalid token specified\+: missing part \#} =\texorpdfstring{$>$}{>} this probably means you are missing a dot ({\ttfamily .}) in the token
\item {\ttfamily Invalid token specified\+: invalid base64 for part \#} =\texorpdfstring{$>$}{>} the part could not be base64 decoded (the message should contain the error the base64 decoder gave)
\item {\ttfamily Invalid token specified\+: invalid json for part \#} =\texorpdfstring{$>$}{>} the part was correctly base64 decoded, however, the decoded value was not valid JSON (the message should contain the error the JSON parser gave)
\end{DoxyItemize}\hypertarget{README.md_autotoc_md11162}{}\doxysubsubsubsubsection{\texorpdfstring{Use with Type\+Script}{Use with Type\+Script}}\label{README.md_autotoc_md11162}
The return type of the {\ttfamily jwt\+Decode} function is determined by the {\ttfamily header} property of the object passed as the second argument. If omitted (or set to false), it\textquotesingle{}ll use {\ttfamily Jwt\+Payload}, when true it will use {\ttfamily Jwt\+Header}. If needed, you can specify what the expected return type should be by passing a type argument to the {\ttfamily jwt\+Decode} function.

You can extend both {\ttfamily Jwt\+Header} and {\ttfamily Jwt\+Payload} to include non-\/standard claims or properties.
